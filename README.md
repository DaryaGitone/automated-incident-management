# Automated Incident Management System

End-to-end no-code automation that processes support requests, assigns priority, and notifies clients in real time.

---

## 📌 Overview
This project automates the handling of support requests — from user submission to structured tracking and client communication.

It acts as a lightweight ticketing system that reduces manual work and improves response time.

---

## ⚠️ Problem
Manual triage of support requests is slow and prone to errors. Without a structured system:
- Requests can be lost
- Response time increases
- Client experience suffers

---

## ✅ Solution
Built an automated workflow using Make that:
- Processes incoming requests
- Assigns priority (High / Low)
- Sends instant email notifications
- Logs structured data in Google Sheets

---

## 🔄 Workflow
1. Trigger: Google Forms submission
2. Data processing
3. Routing (High / Low priority)
4. Email notification via Gmail
5. Data logging in Google Sheets
